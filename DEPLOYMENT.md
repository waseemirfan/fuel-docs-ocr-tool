├── sample_delivery_sites.csv     # Example data for testing

│   ├── app/                      # Source code

│   ├── requirements.txt          # Python dependencies
│   ├── .env.example              # Configuration template

## Deployment Checklist

### For Shipping to Client

- [ ] **Verify frontend is built**
  ```bash
  ls -la frontend/dist/
  # Should contain: index.html, assets/
  ```

- [ ] **Test the startup script locally**
  ```bash
  ./start.sh
  # App should launch at http://localhost:5173
  ```

- [ ] **Remove temporary files**
  ```bash
  # Clean build artifacts
  rm -rf backend/app.db
  rm -rf backend/uploads/*
  rm -rf venv/
  ```

- [ ] **Create distribution archive**
  ```bash
  # macOS/Linux
  cd /Users/sirfan19/Downloads/Personal\ Usecases/
  tar -czf FuelDocs-OCR-Tool.tar.gz "OCR tool"
  
  # Windows (from 7-Zip or WinRAR)
  # Right-click → 7-Zip → Add to Archive → FuelDocs-OCR-Tool.zip
  ```

- [ ] **Document client requirements**
  - Python 3.10+
  - 4GB RAM minimum
  - Internet connection (for LLM)

## Installation on Client Machine

### Client receives:
1. Extraction script or zip file
2. Link to this deployment guide
3. Instructions to run: `./start.sh` (Mac/Linux) or `start.bat` (Windows)

### Client does:
1. Extract the archive
2. Open terminal in folder
3. Run the startup script
4. Wait ~30 seconds for services
5. Open http://localhost:5173
6. Configure LLM provider in Settings
7. Start uploading documents

## Customization Options

### Branding
Edit `frontend/src/pages/Dashboard.tsx` to add custom logo/footer

### API Port
Edit `backend/.env`:
```env
API_PORT=8000  # Change if needed
```

### Default LLM Provider
Edit `backend/.env`:
```env
LLM_PROVIDER=gemini  # or: openrouter, ollama
```

### Extraction Fields
Edit `backend/app/services/llm_service.py` - `EXTRACTION_PROMPT`

## Client Support

### Common Issues
1. **"Python not found"** → Install from python.org
2. **"Port in use"** → Change port in .env
3. **"API key error"** → Check key format in Settings
4. **"Slow extraction"** → Normal if using free tier

### Logs Location
- Backend logs: Terminal output (when running start script)
- Database: `backend/app.db`
- Uploads: `backend/uploads/` folder

## Updates & Maintenance

### For Client to Update Code
```bash
# Stop the app (Ctrl+C in terminal)
# Then:
cd backend
git pull origin main  # if using Git
pip install -r requirements.txt -U
./start.sh
```

### For You to Push Updates
1. Make changes to backend/frontend
2. Test locally: `./start.sh`
3. Rebuild frontend: `npm run build`
4. Create new archive: `tar -czf FuelDocs-OCR-Tool-v2.tar.gz "OCR tool"`
5. Send to client with update notes

## Production Considerations

### For Scaling Beyond Single User
- Deploy backend separately (AWS/Azure/GCP)
- Use external database (PostgreSQL)
- Serve frontend from CDN
- Add authentication & multi-user support
- See: `backend/app/models/user.py` for future expansion

### Security
- ✓ .env file contains secrets (add to .gitignore)
- ✓ Database stored locally (no backup to cloud by default)
- ✓ CORS restricted to localhost
- ✓ Upload folder validated (images only)

For production deployment, consider:
- Adding user authentication
- Using environment variable secrets manager
- Setting up database backups
- Enabling HTTPS
- Adding rate limiting
